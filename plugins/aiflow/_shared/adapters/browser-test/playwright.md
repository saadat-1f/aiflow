# Adapter — browser-test: Playwright

Active when `capabilities.automation.tool == "playwright"` (web projects). Pairs with
`playbook/visual-diff.md` for interactive-state/visual checks.

**Load tools:** `ToolSearch "+playwright navigate"` (fallback `"playwright"`). Use the Playwright
**MCP** for live interaction — do NOT shell out to `npx playwright` for the interactive pass.

**Config:** base URL = `capabilities.runtime.config.url` (NOT hardcoded). Spec output =
`conventions.specDir`/`conventions.specExt`; spec language = `conventions.specLanguage`.

**Verbs (MCP):** navigate, click, fill_form/type, select_option, resize, snapshot, take_screenshot,
wait_for, evaluate, network_requests.

**Waits (generic):** after navigation, wait for network idle + DOM content loaded. For
async/conditionally-rendered content, wait for the element's visible/hidden state. For animations,
wait for the transition to settle. (Stack-specific waits — a charting lib's CDN ready signal,
currency parsing, analytics-layer interception — belong in the project's own `test-patterns`
generated by `codebase-context`, not here.)

**Selector priority:** `data-testid` → role (getByRole) → text (getByText) → label → CSS (last
resort; class names may be build-hashed — use partial `[class*="…"]` matches).

**Spec generation:** emit one test per matrix row, grouped by category, in
`conventions.specLanguage` to `<specDir>/<idfile>.<specExt>`; use relative `page.goto('/route')`
with baseURL from config; assertions reflect what was verified live (cite matrix trace IDs).
